## Research Program Function Assignment Map

When a research commission is active, the following functions are typically engaged:

```
Stage 2 — Source Acquisition
└── CF-02 Research Intelligence (primary)

Stage 3 — Independent Investigation (PARALLEL — no cross-function communication)
├── CF-02 Research Intelligence     → Evidence acquisition
├── CF-03 Comparative Intelligence  → Framework mapping
├── CF-04 Educational Intelligence  → Conceptual vocabulary
├── CF-05 Computational Intelligence → Formal analysis (if assigned)
└── CF-06 Executive Intelligence    → Executive framing (draft)

Stage 4 — Comparative Analysis
└── CF-03 Comparative Intelligence (leads; now sees CF-02 outputs)

Stage 5 — Institutional Synthesis (PARALLEL outputs)
├── CF-01 Institutional Architecture → Canonical Synthesis; Master Index
└── CF-06 Executive Intelligence    → Executive Briefing; visualizations

Stages 6–9 — Integration
└── CF-01 Institutional Architecture (coordinates; integrates all outputs)
└── CF-09 Memory Intelligence       (commits all canonical outputs to Vault)
```

**Independence Protocol:** No inter-function communication is permitted during Stage 3. Each function investigates the research question independently before Stage 4 comparative analysis begins.

## Engine Succession Policy Summary

Engines are evaluated on 10 criteria:

| # | Criterion | What It Measures |
|---|-----------|-----------------|
| 1 | Capability | Raw ability to perform core function tasks |
| 2 | Reliability | Output consistency and operational availability |
| 3 | Cost | Financial cost relative to value delivered |
| 4 | Research Quality | Evidence acquisition, source evaluation, uncertainty acknowledgment |
| 5 | Reasoning Quality | Multi-step reasoning, framework analysis, nuanced judgment |
| 6 | Engineering Quality | Code generation, system design, security, documentation |
| 7 | Long Context | Coherence and accuracy across very long context windows |
| 8 | Tool Use | Reliable tool call execution and output interpretation |
| 9 | Governance Alignment | Provider transparency, ToS compatibility, safety posture |
| 10 | Future Compatibility | Engine trajectory; provider stability; architectural forward-compatibility |

Scores are 1–5 per criterion, weighted by function, averaged across evaluators. Full scoring anchors and weighting matrix: [[Engine Succession Policy]].

**Succession recommendation thresholds:**
- Challenger scores 0.5+ higher than incumbent → succession recommended
- Challenger scores 0.3–0.5 higher → Cognitive Council deliberates
- Challenger scores <0.3 higher → no change; monitor
