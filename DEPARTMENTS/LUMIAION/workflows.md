# Workflows

## Decision Routing

Input: request, blocker, project question, or approval need.

Steps:

1. Classify the request by department.
2. Check whether Founder approval is required.
3. Route operational work to the correct department.
4. Route strategic decisions to approval.
5. Record final decision in the relevant index or log.

Output: routed task, approval request, or executive summary.

## Cross-Department Alignment

Input: conflicting department outputs.

Steps:

1. Identify the conflict.
2. Compare source documents.
3. Draft resolution options.
4. Request Founder approval only if the decision changes strategy.

Output: alignment note or approval request.
