# Canonical Approval Record

## Purpose

The single log every approval decision is recorded in, implementing `../GOVERNANCE/approval-authority-matrix.md`'s canonical model. Where `CPOS/APPROVAL_GATES.md` continues to track gate status for active production work, this log is the durable record of the decision itself — CPOS is the working queue, this is the permanent register.

## Record Format

Each entry uses the fields defined in `../GOVERNANCE/approval-authority-matrix.md`: Decision ID, Subject, Requester, Approving Authority, Date, Scope, Status (`approval_status` vocabulary), Conditions, Linked Files, Revision Impact. Template: `../TEMPLATES/approval-record.template.md`.
