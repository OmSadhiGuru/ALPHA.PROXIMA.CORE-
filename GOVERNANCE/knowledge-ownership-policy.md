# Knowledge Ownership Policy

## Purpose

State who owns, or is credited for, each kind of material moving through this repository's knowledge pipeline — and separate institutional knowledge ownership from personal data governance, which follows different rules entirely.

## Ownership by Kind

| Kind | Ownership / Attribution |
|---|---|
| Raw sources | Remain attributed to their original creator. Never reattributed to Alpha Proxima. |
| Source Records | Institutional record; the *description* is Alpha Proxima's, the *source* it describes remains externally attributed. |
| Extraction Notes | Institutional; traceable to the Source Record and, through it, to the original creator. |
| Evidence Classification | Institutional judgment, applied by the extractor/classifier, not a claim of ownership over the source. |
| Founder Synthesis | **OSG synthesis may be owned as an original framework** — the act of synthesis produces something new — **while preserving traceability** to every contributing source. |
| Knowledge Objects | Institutional (Alpha Proxima), carrying forward all source attribution from the stages that produced them. |
| Research Notes / Studies | Institutional, with external sources attributed per the Source Record they draw on. |
| Public Content, Courses, Books | Institutional (OSGMETAPHYSICS-published), subject to the same attribution-preservation rule for any external material they draw on. |
| User-generated data | Belongs to the user who generated it. Institutional use requires the same consent/governance boundary as personal tracking data, below. |
| Personal tracking data (e.g., gym logs, health notes, dreams — `../TEMPLATES/designed-founder-brief.template.md`) | **Governed separately from institutional knowledge.** Personal data is not a Knowledge Object and does not enter `../KNOWLEDGE/` directly; it may inform institutional knowledge only through an explicit synthesis step that strips or generalizes personal identifiers. |
| Anonymized research data | Institutional, provided anonymization is genuine and documented — the anonymization method is itself recorded in the derived Knowledge Object's Source Material. |

## Core Rules

1. **Original external sources remain attributed to their creators.** This rule is absolute and does not weaken as material moves further through the pipeline (Extraction → Classification → Synthesis → Knowledge Object).
2. **OSG synthesis may be owned as an original framework while preserving traceability.** Ownership of the synthesis is not ownership of the sources beneath it.
3. **User personal data remains governed separately from institutional knowledge.** It does not inherit institutional-knowledge ownership rules by default, and does not enter `../KNOWLEDGE/` without an explicit, documented synthesis/anonymization step.
