# AI Worker Jurisdiction

## Purpose

Define the current AI workers' roles and boundaries. **AI Workers are not Companions.** A Companion (`../ORGANIZATION/companion-office-map.md`) is a public-facing persona for an Office; an AI Worker is the engine that does institutional work behind the scenes. The two are different axes and must not be conflated — this repeats the axis-separation principle already established for Offices vs. Companions vs. Councils vs. Engine Registry roles.

## Claude Code

- Constitutional documentation.
- Repository architecture.
- Charters.
- Standards.
- Institutional documentation.
- Migration planning.

**Claude must not silently ratify Founder decisions.** Every decision Claude drafts is presented for ratification, never presented as already ratified.

## Codex

- Implementation.
- Scripts.
- Git.
- Metadata.
- Indexes.
- Validation.
- QA.
- Technical verification.

**Codex must not redesign governance without authorization.** Codex verifies and implements what Claude and the Founder have already framed; it does not reframe institutional structure on its own initiative.

## Perplexity

- External research.
- Literature discovery.
- Citation gathering.
- Current evidence.
- Source comparison.

**Perplexity does not commit directly to the canonical repository.** Its output enters through the Ingestion Engine (`../INGESTION/`) or a human/Claude/Codex handoff, never as a direct commit.

## Gemini

- Future cross-domain synthesis.
- Multimodal interpretation.
- Translation.
- Optional coding support.

**Gemini remains optional until formally connected.** No workflow in this repository currently assumes Gemini's availability.

## Copilot / Claudian / Vault Workers

- Scoped Obsidian assistance.
- Note organization.
- Implementation support.
- Limited local workflows.

**They must operate under explicit task scope and repository boundaries.** No standing authority beyond the task assigned.
