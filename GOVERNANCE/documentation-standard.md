# Documentation Standard

## Required Frontmatter

Every durable knowledge document must support:

```yaml
---
title:
id:
department:
domain:
type:
status:
version:
created:
updated:
source:
tags:
related:
owner:
---
```

## Required Body Sections

Use sections appropriate to the document type. At minimum, durable documents should define:

- Purpose.
- Scope.
- Inputs.
- Outputs.
- Owner.
- Status.
- Related folders or documents.

## Writing Rules

- Use precise technical language.
- Avoid decorative phrasing.
- State assumptions explicitly.
- Link related documents by relative path.
- Keep operational instructions separate from narrative context.
