# OSG 30 Day Implementation Roadmap

## Purpose

Define the next 30 days required to prepare OSG for launch.

## Week 1 - Foundation

- Create separate OSG GitHub repository.
- Add README, `.gitignore`, `.env.example`, and GitHub PR template.
- Create Notion HQ with Courses, Clients, Content, Media, Automations, Operations, and Metrics databases.
- Confirm launch offer, course/client path, and primary CTA.
- Create landing page outline.
- Define naming conventions across GitHub, Notion, media, and content.

## Week 2 - Offer and Systems

- Build landing page or launch page.
- Configure Calendly booking flow.
- Configure Stripe product, payment link, or checkout path.
- Configure email list and welcome sequence.
- Create client onboarding checklist.
- Create course production checklist.
- Connect core manual workflows before automating.

## Week 3 - Content and Delivery

- Produce first launch content batch.
- Build course or client delivery skeleton.
- Create media asset registry.
- Test booking-to-follow-up flow.
- Test payment-to-onboarding flow.
- Create weekly operations dashboard.
- Run first internal launch rehearsal.

## Week 4 - Launch Readiness

- Complete QA pass on landing page, payment, booking, email, and onboarding.
- Publish launch content.
- Activate minimum automations.
- Review metrics daily.
- Record blockers and decisions.
- Create post-launch improvement backlog.
- Tag repository milestone `launch-v1.0`.

## Launch Definition of Done

- A prospect can understand the offer.
- A prospect can book or buy.
- Payment and booking notifications work.
- Follow-up and onboarding are documented.
- Content pipeline is active.
- Client/course delivery has a clear home.
- No OSG public business data is mixed with Alpha Proxima infrastructure.
